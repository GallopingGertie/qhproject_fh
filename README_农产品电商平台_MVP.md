# 某省农产品电商平台（MVP 第一阶段）README

## 1. 项目简介

本项目是一个面向“某省特色农产品销售”的平台型电商系统，目标是在第一阶段先完成**最小可运行闭环**，快速支撑平台上线、商户入驻、商品售卖、订单履约与基础结算。

本阶段不追求大而全，而是围绕以下主线流程建设：

**商户入驻申请 → 平台审核 → 商户创建商品 → 平台审核商品 → 用户浏览下单 → 用户支付 → 商户发货 → 用户查询物流 → 用户确认收货 / 发起退款 → 平台基础结算**

该版本聚焦两类前端：

- 用户端 App / 小程序
- 后台 Web（同时承载平台端和商户端）

## 2. 第一阶段建设目标

### 2.1 核心目标

第一阶段只解决三个问题：

1. 商户可以申请入驻，并在审核通过后发布商品
2. 用户可以浏览商品、下单、支付、查看订单和物流
3. 平台可以审核商户和商品、管理订单、处理退款、做基础结算

### 2.2 项目边界

第一阶段**只做基础交易版**，暂不建设以下能力：

- 会员等级
- 积分体系
- 拼团/秒杀
- 分销裂变
- 店铺装修高级版
- 客户标签和私域运营
- 复杂 BI 报表
- 自动风控引擎
- 冷链溯源
- 商户独立 App

### 2.3 成功标准

系统上线后，至少应满足：

- 商户能完成入驻申请与资料提交
- 平台能审核商户并启用账号
- 商户能创建商品并提交审核
- 平台能审核商品并控制上架/下架
- 用户能完成下单支付
- 商户能完成发货并录入物流单号
- 用户能在客户端查看物流与订单状态
- 用户能申请退款
- 平台能查看订单金额和商户可结算金额

## 3. 项目定位与业务模型

### 3.1 项目定位

这是一个**平台型农产品电商系统**，不是单店商城。

系统中存在三类核心角色：

- **平台管理员**：负责审核、管理、监管、结算
- **商户**：负责入驻、上架商品、发货、查看经营数据
- **用户**：负责浏览商品、下单、支付、查看订单、申请退款

### 3.2 核心业务模型

平台统一提供：

- 招商入驻能力
- 商品审核能力
- 订单和退款管理能力
- 支付与基础结算能力
- 后台数据统计能力

商户只负责：

- 提交入驻资料
- 管理店铺基础信息
- 维护商品信息
- 接单与发货
- 查看基础财务数据

用户只负责：

- 浏览商品
- 购买商品
- 查看订单与物流
- 发起退款

## 6. 核心业务主线逻辑

### 6.1 主线流程

```text
商户提交入驻申请
→ 平台审核商户资料
→ 审核通过后开通商户后台权限
→ 商户创建商品并提交审核
→ 平台审核商品
→ 审核通过后商品上架
→ 用户浏览商品并下单
→ 用户完成支付
→ 商户后台接单并发货
→ 用户客户端查看物流
→ 用户确认收货 / 或发起退款
→ 平台进行基础结算
```

### 6.2 平台必须具备的最小管控能力

即使是 MVP，也必须保留以下 5 项“平台控制能力”：

1. **商户入驻审核**：商户不能自动开店
2. **商品审核**：商户商品不能直接对外售卖
3. **平台统一收款**：支付走平台，不走商户私收
4. **基础抽成结算**：平台可按规则统计应结算金额
5. **平台可处理退款**：退款流程平台可见、可处理、可介入

## 11. 核心业务规则

## 11.1 商户入驻规则

- 商户必须先注册并提交资料
- 审核未通过前不能经营
- 平台可以驳回并要求重新提交资料
- 平台可以手动停用商户账号

## 11.2 商品审核规则

- 商户新增商品后必须提交审核
- 审核通过后商品方可上架
- 平台可下架任意商品
- 驳回时需记录驳回原因

## 11.3 订单规则

- 用户提交订单后生成待支付订单
- 支付成功后订单变更为待发货
- 商户发货后订单变更为待收货
- 用户确认收货后订单变更为已完成
- 已退款订单不参与商户可提现余额

## 11.4 退款规则

- 用户可对符合条件的订单发起退款
- 平台可审核退款申请
- 退款成功后订单进入退款完成状态
- 已退款金额需冲减商户结算金额

## 11.5 结算规则

- 平台统一收款
- 平台按设定比例抽成
- 商户结算基于已完成订单进行统计
- 退款订单需从结算中扣减
- 提现需平台审核

## 14. 技术实现建议

## 14.1 技术选型

### 前端

- 用户端：`uni-app` 或微信小程序原生开发
- 后台 Web：`Vue 3 + Element Plus`

### 后端

- `Java + Spring Boot`

### 数据库与基础设施

- `MySQL`：主业务数据
- `Redis`：缓存、登录态、简单状态加速
- `OSS / 对象存储`：商品图片、资质文件

### 第三方接口

- 微信支付
- 支付宝（按需要）
- 快递查询接口
- 短信服务（按需要）

## 14.2 架构建议

第一阶段采用：

**单体应用 + 模块化分层架构**

原因：

- 开发速度快
- 部署简单
- 成本更低
- 适合 MVP 验证

不建议第一阶段直接上微服务。

## 18. 权限设计建议

第一阶段采用简单 RBAC 即可。

### 角色

- 平台管理员
- 商户
- 用户

### 权限原则

- 用户只能访问自己的订单、地址、退款记录
- 商户只能访问自己店铺的商品、订单、财务
- 平台管理员可以访问全平台数据

请基于本 README 生成一个“农产品电商平台 MVP”项目，包含两个前端和一个后端：
